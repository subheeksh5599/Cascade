What Cascade Does

Most financial applications only move money. Cascade defines how money behaves after deposit. Builders define a directed acyclic graph where each node is a FlowVault routing rule. The keeper automaton walks the graph in topological order — configuring routing rules, executing deposits, and cascading funds to downstream nodes — until every leaf node receives its allocation.

FlowVault Integration

Cascade uses all three FlowVault primitives in combination, orchestrated by a DAG engine. The keeper walks nodes in topological order, calling the FlowVault SDK at each step.

🔒

Lock

Time-bound escrow at each node. Funds are locked until a specific Stacks block height. Used for runway reserves, milestone locks, and vesting schedules.

⚖

Split

Instant routing to recipient addresses. A percentage of incoming funds is sent to a designated Stacks principal — salaries, DAO contributors, agent commissions.

💧

Hold

Liquid pass-through. The remaining balance after Lock and Split stays available and cascades to downstream child nodes, powering the recursive flow.
